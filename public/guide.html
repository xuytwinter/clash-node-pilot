Clash Node Pilot

使用教程

按当前代理组和地区持续选择稳定、低延迟的可用节点。

01

首次运行

保持代理客户端与代理核心运行，然后启动 Node Pilot：

npm start

打开 http://127.0.0.1:3210。顶部显示“已连接”后，程序已经读取到本机控制器。

02

手动优选

  1. 选择在线的代理客户端。
  2. 选择需要切换的手动代理组。
  3. 选择目标地区。
  4. 保持“测速后自动切换”开启，点击“开始测速并优选”。

结果按照综合健康评分排列。绿色结果为本轮优选节点；失败节点不会中断其他节点测速。

03

自动轮询

在“自动测速设置”中选择每隔几分钟测速一次，默认是 3 分钟。只要当前地区还有健康节点，程序就不会跨地区。

当前地区测速→达到切换阈值→切换最快节点

只有当前地区全部失败，才使用第二个探测地址测试其他地区。所有地区都失败时保持原配置。

人工操作保护

检测到你在代理客户端手动切换节点后，为避免覆盖你的选择，自动测速与切换默认暂停 15 分钟。也可以点击“锁定 15 分钟”立即暂停，或开启“仅监控，不切换”。

04

参数设置

参数默认值作用
自动测速间隔3 分钟选择每隔几分钟执行一次自动测速与优选。
自动切换至少快多少25 ms新节点至少快到这个幅度才切换，防止频繁跳动。
每个节点测速次数2多次探测后取中位数，减少偶发抖动。
手动切换后暂停自动优化15 分钟尊重用户刚完成的人工选择。
测速超时5000 ms单次探测的最长等待时间。

设置、历史、锁定状态和节点健康记录保存在本地 data/state.json，重启后继续生效。

05

客户端支持

客户端状态
Clash Verge Rev完整支持
Clash for Windows完整支持
其他 Clash/Mihomo 客户端通过 CLASH_CONFIG 指定配置
v2rayN 7.x只读检测

v2rayN 当前可显示节点数量、当前节点和历史延迟。由于上游没有外部节点切换 API，Node Pilot 不会修改其数据库或强制重启 Xray。

06

开机启动

首次启动控制台后，点击页面右上角的“开机启动”。按钮显示“已开启”后，以后登录 Windows 不再需要手动执行 npm start。

登录后看门狗会等待网络与 DNS 可用，验证 Mihomo 控制器，必要时恢复 Clash Verge 核心，再启动 Node Pilot。即使 Clash Verge 窗口被 Windows 恢复但核心没有启动，也会自动修复。

命令行安装：

powershell.exe -ExecutionPolicy Bypass -File .\install-pilot-autostart.ps1

管理员任务计划方案还会每 2 分钟检查一次：

powershell.exe -ExecutionPolicy Bypass -File .\install-autostart-admin.ps1

卸载启动项：

powershell.exe -ExecutionPolicy Bypass -File .\uninstall-pilot-autostart.ps1
07

故障处理

Clash Verge 窗口已打开但无法代理
看门狗会通过控制器接口判断核心是否健康并自动恢复。诊断记录位于 data/startup-watchdog.log。
显示控制器离线
确认代理客户端和核心均在运行，并检查外部控制器是否启用。
地区没有节点
节点名称可能不包含可识别地区。可编辑 regions.json 添加机场的命名规则。
没有切换到最低延迟
程序会应用切换阈值和历史失败率惩罚。小幅改善会保持当前节点。
页面与客户端不同步
页面每 15 秒自动刷新，也可点击右上角刷新按钮立即同步。
自动任务没有运行
检查 Windows 任务计划程序中的 Clash Node Pilot Startup、Clash Node Pilot Watchdog 和 Clash Node Pilot Optimizer。
